Driver communication via a parallel interface

Computer System Design

I have written a driver to receive a vector of K bytes through a serial device, using the interrupt mechanism. The received elements are stored only if the first two bits of the byte are high, while otherwise they are not stored. At the end of the reception, if less than K characters have been stored, the receiving system must send an appropriate message (e.g. a specific byte) to the transmitting system via a properly connected serial device, and make sure that the message has been read before disable reception interruptions
